    1. What is Semantic HTML? 

A. Semantic HTML are elements that clearly defines the content of which it displays.

    2. What is HTML used for? 

A. HTML is used to structure a website.

    3. What is an attribute and where do we put it? 

A. Attributes are defined in opening html tags, and give more information about the element.

    4. What is the h1 tag used for? How many times should I use it on a page?

A. The h1 tag is used to describe the main content of the page, and only should be used once.

    5. Name two tags that have required attributes

A. the <a> tag and <img> tag.

    6. What do we put in the head of our HTML document? 

A. We put metadata related elements in the head section.

    7. What is an id?

A. id is an attribute we use to identify specific tags, usually to manipulate later in CSS> 

    8. What elements can I add an id to? 

A. You can add an ID to any element.

    9. How many times can I use the same id on a page? 

A. You may only use the ID once.

    10. What is a class? 

A. A class is like ID but can be used by multiple HTML elements unlike id.

    11. What elements can I add a class to? 

A. You can add the class attribute to any HTML element.

    12. How many times can I use the same class on a page? 

A. As many as needed.

    13. How do I get my link to open in a new tab?

A. By including "target='_blank'" in the <a> tag.

    14. What is the alt attribute in the image tag used for?

A. The alt attribute gives alternate text in case of the element not working.    

    15. How do I reference an id?

A. By using #id_example {}

    16. What is the difference between a section and a div

A. A section usually is a much larger box/area containing multiple div with elements,
   whereas divs are smaller sections usually grouping a few elements together.

    17. What is CSS used for?

A. CSS is used to take the HTML structure and add style.

    18. How to we select an element? Example - every h2 on the page

A. By doing "h2 {}"

    19. What is the difference between a class and an id? - Give me an example of when I might use each one

A. You would use id when targeting a specific element where you would use class when targeting multiple elements.

    20. How do we select classes in CSS?

A. by doing " .classexample {}"

    21. How do we select a p element with a single class of “human””?

A. .human p {}

    22. What is a parent child selector? When would this be useful? 

A. It's used to only manipulate children elements from a specific parent element, and would be great for more in depth customization.

    23. How do you select all links within a div with the class of sidebar?

A. .sidebar a {}

    24. What is a pseudo selector?

A. A pseudo selector connects to elements and only shows changes within the define state.

    25. What do we use the change the spacing between lines?

A. The <br> tag

    26. What do we use to change the spacing between letters?

A. Use text-spacing

    27. What do we use to change everything to CAPITALS? lowercase? Capitalize?

A. Use text-transform

    28. How do I add a 1px border around my div that is dotted and black?

A. .example {
    border: 1px dotted black;
}

    29. How do I select everything on the page? 

A. Use * {}

    30. How do I write a comment in CSS?

A. Use /* */

    31. How do I find out what file I am in, when I am using the command line? 

A. Use pwd

    32. Using the command line - how do I see a list of files/folders in my current folder?

A. Use li

    33. How do I remove a file via the command line? Why do I have to be careful with this? 

A. Use git rm, but be careful because it deletes the file and can lead to accidents.

    34. Why should I use version control? 

A. To have a working version at all times while multiple developers have their own version to work on and merge into the master version.

    35. How often should I commit to github?

As often as possible

    36. What is the command we would use to push our repo up to github? 

A. push origin firstname-lastname

    37. Walk me through Lambda's git flow.

A.
    1. Fork a repo from github
    2. Clone using git clone in git bash
    3. Branch bt using the git checkout command
    4. Edit repo
    5. Add by using git add .
    6. Commit -m "your message"
    7. Push to github with git push origin branch-Name
    8. Pull request through github
